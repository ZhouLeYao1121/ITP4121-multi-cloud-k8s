# ITP4121-multi-cloud-k8s
Deploy and manage Multi Cloud Kubernetes with Terraform

Project Overview
This project demonstrates the implementation of Infrastructure as Code (IaC) using Terraform to automate the deployment of infrastructure across public cloud platforms.
The primary focus of this project is to design and code a modular, scalable, and secure cloud infrastructure for a containerized application. Due to environmental limitations encountered during the deployment phase (explained below), the project was finalized to use Azure as the main cloud platform, while maintaining a complete multi-cloud architecture design.

1. Complete Terraform Codebase
Wrote clean, syntactically correct HCL code.
Implemented core Azure resources: Resource Group, Virtual Network, Subnets, and AKS (Kubernetes cluster).
Designed a secure network architecture with private subnets for application and data tiers.
2. Git Version Control
Established a professional GitHub repository.
Managed the entire development lifecycle with git (commit, push, reset).
Ensured code integrity and version history are fully preserved.
3. Multi-Cloud Architecture Design
Created a detailed architecture blueprint supporting Azure + GCP hybrid infrastructure.
Designed VPC/Network topology, security rules, and cluster configurations.

Technologies & Resources Used
Terraform: For infrastructure coding and provisioning.
Azure: As the primary deployment platform (Resource Group, VNet, AKS).
GitHub: For version control and project collaboration.
Cloud Shell (GCP): For development and testing environment.
